### 一、安全中心概述

系统安全中心是**安全智能体团队**的核心工作界面，由**安全合规部**的智能体团队负责。主脑（CEO）管理安全团队，但不直接执行具体安全任务。安全团队包括：安全主管、安全工程师、合规审计师、数据隐私官等智能体。

| 使用角色 | 权限范围 |
|---------|---------|
| 老板 | 查看安全态势，审批重大安全决策 |
| CEO/主脑 | 管理安全团队，接收安全报告，决策重大安全事项 |
| CTO | 查看安全态势，协调技术安全 |
| 安全主管 | 管理全部安全功能，分配安全任务 |
| 安全工程师 | 执行安全扫描、漏洞修复、攻击防护 |
| 合规审计师 | 执行合规检查、审计日志审查 |
| 数据隐私官 | 管理数据隐私、处理隐私请求 |

## 安全智能体团队职责分工

| 智能体 | 层级 | 岗位 | 核心职责 |
|--------|------|------|---------|
| 安全主管 | L4 | 安全负责人 | 统筹安全策略，管理安全团队，向主脑汇报 |
| 安全工程师A | L5 | 威胁防护专家 | 实时威胁监控，WAF/DDoS防护，攻击溯源 |
| 安全工程师B | L5 | 漏洞管理专家 | 漏洞扫描与修复，依赖库安全，代码审计 |
| 安全工程师C | L5 | 应用安全专家 | 代码安全审计，SDLC安全，渗透测试 |
| 合规审计师 | L5 | 合规专家 | GDPR/SOC2/ISO27001合规，审计日志审查 |
| 数据隐私官 | L5 | 隐私保护专家 | 隐私数据管理，隐私请求处理，脱敏配置 |
| 实习安全助理 | L6 | 助理 | 日志分析辅助，安全事件初步筛选 |


## 与主脑的协作机制

| 事项类型 | 处理方式 | 升级条件 |
|---------|---------|---------|
| 日常安全监控 | 安全团队自主处理 | 无需升级 |
| 高危漏洞 | 安全主管评估后向主脑汇报 | 影响核心业务时 |
| 重大攻击 | 安全工程师立即处置，同步通知主脑 | 持续攻击超过30分钟 |
| 合规缺口 | 合规审计师提出整改方案，主脑审批资源 | 涉及预算或跨部门协调 |
| 隐私事件 | 数据隐私官启动响应流程，通知主脑 | 涉及用户数据泄露 |
| 团队扩编 | 安全主管向主脑申请 | 人手不足时 |
| 安全策略变更 | 安全主管提案，主脑审批 | 影响业务体验时 |

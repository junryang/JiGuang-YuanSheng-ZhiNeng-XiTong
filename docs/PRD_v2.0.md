# 产品需求文档（PRD）v2.0

## 一、产品目标

纪光元生的长期目标是构建“与人类团队无差别的通用自主智能体系统”。  
该目标按**研究目标（Long-term）**与**生产交付（Release Train）**双轨推进：

- **研究目标**：提升自主规划、反事实推理、长期价值权衡、跨模态协作能力。
- **生产交付**：每个版本只交付可验证、可回滚、可观测、可审计的能力子集。

## 二、v2.0 发布范围（冻结）

### 2.1 In Scope（必须交付）

1. 七层组织（L0-L6）+ 角色权限闭环。  
2. CEO 驱动的任务分解、分配、状态汇总。  
3. 能力注册与调用框架（142项能力基线中的生产就绪子集）。  
4. 安全基线（认证、授权、审计、密钥管理、限流）。  
5. 生产可观测性（日志、指标、告警、追踪）与故障降级。

### 2.2 Out of Scope（v2.0不承诺）

1. 全部 142 项能力“一次性100%生产开通”。  
2. 全外部平台无人工介入全自动运营。  
3. 无条件全自治决策（高风险场景仍需人工审批门禁）。

## 三、发布准入门槛（Go/No-Go）

1. **功能**：核心流程验收通过率达到既定门槛（见 `ACCEPTANCE_CHECKLIST_v1.0.md`）。  
2. **安全**：高危漏洞为 0，关键安全测试通过（见 `SECURITY_REQUIREMENTS_v1.0.md`）。  
3. **性能**：满足分类性能指标；长耗时生成指标按 `PERFORMANCE_METRICS_v1.0.md`。  
4. **运维**：具备故障演练记录、回滚预案、值班机制。  
5. **合规**：第三方平台接入与数据使用满足法务与平台条款。

## 四、通用自主智能体成熟度定义（产品口径）

- **M1 协助型**：可执行明确指令，需频繁人工纠偏。  
- **M2 编排型**：可进行多步骤任务编排与回报。  
- **M3 自主型**：在边界内自主决策、可解释、可回滚。  
- **M4 近人类团队型**：复杂跨域协作接近人类团队表现。  

> v2.0 目标：稳定达到 M2，并在受控场景验证 M3 关键能力。

# 首周剩余两天执行计划 v1.0

## 文件路径

`d:\BaiduSyncdisk\JiGuang\docs\WEEK1_REMAINING_2D_PLAN_v1.0.md`

## Goal

在首周剩余两天内完成 Day3 收口闭环，确保发布前门禁、运维清单与组织节奏全部落地，可安全进入 staging→prod 发布窗口。

## Key Constraints

- 不引入大规模新功能，仅做收口与发布准备
- 任何变更必须通过 `tests/` 与 `day3_gate` 两类回归
- 风险处置优先级：P0 > P1 > 文档优化
- 若出现门禁异常，以“先稳态、后扩展”为原则

## Implementation Plan

### Day4（首周倒数第二天）：发布前收口与演练

1. **门禁复核**
   - 复跑 `python -m pytest -m day3_gate -q --tb=short`
   - 复跑 `python -m pytest tests/ -q --tb=short`
   - 更新回归结果快照到 Day3 报告

2. **发布流程演练（桌面演练 + 小流量模拟）**
   - 按 `D3_RELEASE_PACKAGE_v1.0.md` 作为总入口执行（清单 + Runbook + 记录模板）
   - 使用 `D3_RELEASE_REHEARSAL_RECORD_TEMPLATE_v1.0.md` 记录时间线、命令与异常分支
   - 验证 canary 扩流判定与回滚入口可用
   - 记录演练耗时、阻塞点、修正项

3. **值班排班与升级链路确认**
   - 发布总指挥、后端、安全、产品四方名单确认
   - 升级路径和响应 SLA（15 分钟）确认

4. **风险与文档收口**
   - 复核 `DAY3_RISK_CLOSURE_LIST_v1.0.md`
   - 将新增观察项标注为“阻断/非阻断”

### Day5（首周最后一天）：准生产发布准备与管理汇报

1. **Go/No-Go 评审**
   - 输入：回归结果、抽测报告、风险收口单、发布清单
   - 输出：Go/No-Go 决议 + 条件说明

2. **发布工件锁定**
   - 锁定版本标签、镜像 digest、配置快照
   - 形成发布纪要（含回滚版本号）

3. **执行窗口预检查**
   - 健康检查、关键 API、审计字段可读性确认
   - 观测面板阈值确认（错误率、延迟、拒绝码异常）

4. **管理层摘要输出**
   - 输出“首周总结 + 下周进入正式发布节奏建议”
   - 明确遗留事项负责人和截止时间

## Risks and Fallbacks

- 风险：演练中发现策略或审批链路回归  
  兜底：暂停发布准备，回到修复分支并优先处理 P0
- 风险：值班人员不可用  
  兜底：按预备人员替补，未补齐不进入 Go
- 风险：发布窗口观测噪声过大  
  兜底：延后窗口并先做基线噪声治理

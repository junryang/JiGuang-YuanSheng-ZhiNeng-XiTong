# 生产运行假设与降级策略 v1.0

## 一、目的

明确“真实生产环境”中的外部不确定性、系统边界与降级策略，避免设计文档只描述理想路径。

## 二、关键运行假设

1. 第三方平台 API 可用性与速率限制会波动。  
2. 平台政策、内容审核规则、认证机制可能随时变更。  
3. 大模型响应延迟与成本在高峰期可能显著抖动。  
4. 高风险业务动作（资金、发布、权限变更）默认需要人工确认。

## 三、降级策略矩阵

| 场景 | 主策略 | 降级策略 | 退出条件 |
|------|--------|---------|---------|
| 第三方 API 超时/失败 | 重试+指数退避 | 切备用平台/排队/人工接管 | 连续健康检查恢复 |
| 模型响应超时 | 切换同级模型 | 降级到模板/规则生成 | 延迟恢复到阈值内 |
| 配额耗尽 | 节流+排队 | 延迟执行/审批后加配额 | 额度刷新或扩容完成 |
| 风险动作触发 | 自动拦截 | 人工审批流程 | 审批通过 |

## 四、生产门禁建议

1. 任何“全自动”流程必须具备人工接管入口。  
2. 所有关键路径必须有可观测指标（成功率、延迟、错误码、队列长度）。  
3. 关键配置与密钥必须支持轮换与失效演练。  
4. 发布前必须完成演练：第三方中断、模型不可用、数据库故障三类场景。

## 五、与长期目标的关系

本文件不否定“无差别通用自主智能体”目标；  
它定义的是在达到该目标前，系统如何以工程方式安全迭代并持续在线。

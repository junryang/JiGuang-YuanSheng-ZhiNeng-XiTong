# 智能体成熟度模型 v1.0

## 一、目的

将“与人类团队无差别的通用自主智能体”目标拆解为可评估、可交付、可演进的工程里程碑。

## 二、成熟度分级

| 等级 | 定义 | 关键能力 | 评估方式 |
|------|------|---------|---------|
| M1 协助型 | 执行明确指令 | 基础理解、工具调用、结果回报 | 任务完成率/人工纠偏率 |
| M2 编排型 | 多步骤编排与协作 | 任务分解、跨Agent协作、状态管理 | 端到端流程成功率 |
| M3 自主型 | 边界内自主决策 | 反事实、长期价值权衡、风险控制 | 决策质量+回滚率 |
| M4 近人类团队型 | 跨域复杂任务接近人类团队 | 自主规划、持续学习、稳态协作 | 人机对照基准评测 |

## 三、当前目标映射

- v2.0 生产目标：稳定 M2，受控试点 M3。  
- M4 作为长期研究目标，必须通过阶段性基准评测逐步接近。

## 四、硬约束

1. 任何等级都必须满足安全与合规红线。  
2. 任何自主决策都必须可解释、可审计、可回滚。  
3. 任何新能力上线必须先有离线评测与灰度策略。

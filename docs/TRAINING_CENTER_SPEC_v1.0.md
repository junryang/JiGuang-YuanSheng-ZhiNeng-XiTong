# 培训中心设计规范 v1.0

## 文件路径

`d:\BaiduSyncdisk\JiGuang\docs\TRAINING_CENTER_SPEC_v1.0.md`

## 一、定位

培训中心面向**智能体（及必要时人类学员）**的能力提升，核心对象是「学什么」与「如何学」。  
本规范明确：**在合法、合规前提下，由主脑（L1 CEO）主导在全网发现与筛选适合的课程素材**，再经合规闸门与人工角色编排后进入学习闭环。

界面承载见 `UI_AGENT_MANAGEMENT_CENTER_v1.0.md`「培训中心」区块。

## 二、设计原则

1. **合规优先**：任何素材入库或引用前，必须通过版权/授权、平台服务条款、robots/访问策略与适用法律的检查；**禁止**默认整站抓取受保护付费内容或绕过付费墙。  
2. **主脑发现、分层负责**：主脑负责**广域检索、初筛、排序与推荐**；智能体培训师负责**学习路径、节奏与效果评估**；老板/主脑可对高风险类目启用额外审批。  
3. **可追溯**：每条入库素材保留来源 URL、抓取/收录时间、合规判定依据、责任人（系统或角色），满足 `LAW-05` 审计要求。  
4. **最小必要**：优先采用**官方开放课程、知识共享许可（CC 等）、平台公开免费内容、仅链接不复制**等低侵权风险形态；需全文缓存时须有明确授权或许可证类型记录。

## 三、角色分工

| 角色 | 职责 |
|------|------|
| **主脑（CEO）** | 根据能力缺口与培训需求，调度全网检索与多源聚合；调用知识/合规能力对候选素材评分；输出「候选清单」与合规标签。 |
| **智能体培训师** | 将已批准素材编排为课程/模块；分配学员；跟踪进度与测验；不替代主脑做全网发现（可发起「请主脑补充素材」请求）。 |
| **合规/法务（能力或人工）** | 对灰区素材裁决；对批量入库策略签核；争议时一票否决。 |

## 四、端到端流程

1. **需求输入**：来自能力缺口、任务失败分析、绩效或人工指定（与 `HR-02`、学习类能力对齐）。  
2. **主脑发现**：基于主题与难度，通过搜索引擎、开放课程平台、技术文档站点等**合法渠道**收集候选（依赖 `WEB-02`、`WEB-03`、`KNOW-01` 等）。  
3. **质量与去重**：`KNOW-02` 评估相关性、时效与质量；去重与摘要。  
4. **合规闸门**：`LAW-01` 内容合规；`LAW-03` 版权与知识产权；`LAW-04` 访问合法性（是否允许自动化访问、地域限制等）。未通过者**不得**进入可学习资源库，可降级为「仅外部链接」或丢弃。  
5. **入库与元数据**：写入课程素材库：标题、来源、许可类型、是否仅链接、缓存策略、有效期（部分平台内容会下架）。  
6. **编排与交付**：培训师生成学习路径；学员按计划学习；效果回写能力库/绩效。

## 六、数据与状态（建议字段）

```yaml
# 课程素材（节选）
CourseMaterial:
  id: str
  title: str
  source_url: str
  discovery_job_id: str    # 主脑发起的发现任务
  license_type: enum       # e.g. official_open, cc_by, link_only, internal_only, rejected
  compliance_status: enum  # pending, approved, rejected, link_only
  ceo_recommendation_score: float  # 可选
  trainer_packaged: bool   # 是否已由培训师封装为正式课程
  audit_trail_ref: str      # 指向审计记录
```

## 七、UI 与交互要点

- 培训中心增加 **「主脑素材发现」** 视图：展示发现任务、候选列表、合规状态、一键「请求主脑再找」。  
- **课程库**列表展示：**来源类型**（官方/CC/链接等）、**合规状态**，避免与未审核素材混淆。  
- 对仅链接类课程，学习动作跳转到站外时提示用户遵守目标站点规则。
